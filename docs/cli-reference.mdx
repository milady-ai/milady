## Starting Milaidy

```bash
milaidy                    # start interactive TUI (default command)
milaidy tui                # same thing â€” interactive terminal chat
milaidy tui -m <model>     # start TUI with a specific model (e.g. anthropic/claude-sonnet-4-20250514)
milaidy start              # start the agent runtime (API server only, no TUI)
milaidy run                # alias for start
```

## Setup & Config

```bash
milaidy setup              # initialize ~/.milaidy/milaidy.json and agent workspace
milaidy setup --workspace <dir>  # specify custom workspace directory
milaidy configure          # display configuration guidance and common env vars
milaidy config get <key>   # read a config value (supports dot-notation)
milaidy config path        # print the resolved config file path
milaidy config show        # display all config values grouped by section
milaidy config show --all  # include advanced/hidden fields
milaidy config show --json # output raw JSON
```

## Dashboard & UI

```bash
milaidy dashboard          # open web UI in browser
milaidy dashboard --port 3000  # custom port
milaidy dashboard --url <url>  # open a specific URL directly
```

## Models

```bash
milaidy models             # list configured model providers (checks env vars)
```

## Plugins

```bash
milaidy plugins list               # list all plugins from the registry
milaidy plugins list -q <keyword>  # filter by name or keyword
milaidy plugins search <query>     # search the plugin registry
milaidy plugins info <name>        # show detailed info about a plugin
milaidy plugins install <name>     # install a plugin (supports name@version)
milaidy plugins uninstall <name>   # uninstall a user-installed plugin
milaidy plugins installed          # list plugins installed from the registry
milaidy plugins refresh            # force-refresh the registry cache
milaidy plugins test               # validate custom drop-in plugins
milaidy plugins config <name>      # show plugin configuration
milaidy plugins config <name> -e   # interactive plugin config editor
milaidy plugins add-path <path>    # register a plugin search directory
milaidy plugins paths              # list all plugin search directories
milaidy plugins open [name-or-path]  # open a plugin directory in your editor
```

## Updates

```bash
milaidy update                     # check for and install updates
milaidy update --check             # check only, don't install
milaidy update --channel <ch>      # switch release channel (stable, beta, nightly)
milaidy update --force             # bypass interval cache
milaidy update status              # show versions across all channels
milaidy update channel [name]      # view or change the release channel
```

## Misc

```bash
milaidy --version          # version check
milaidy --help             # help
```

## TUI (Terminal UI)

When running, Milaidy shows a live interactive terminal chat interface with a status bar, chat area, and input editor.

## TUI Hotkeys

| Key | Action |
|-----|--------|
| `Enter` | send message |
| `Shift+Enter` | insert newline |
| `Ctrl+P` | open model selector |
| `Ctrl+E` | expand/collapse tool output |
| `Ctrl+C` | cancel current operation or quit |

## TUI Slash Commands

The following slash commands are available in the TUI input:

| Command | Action |
|---------|--------|
| `/model` | switch model (opens selector or `/model provider/id`) |
| `/models` | alias for `/model` |
| `/clear` | clear chat history |
| `/help` | show help |
| `/exit` | quit |
| `/quit` | alias for `/exit` |

## Headless / Server Mode

Don't want the TUI? Use the `start` command for API-server-only mode:

```bash
milaidy start
```

Logs go to `~/.milaidy/logs/`. Daemonize with your favorite process manager.
